What this website does

1
Developers create an account
Register with name, email and password. Password is hashed with bcrypt before saving. A JWT token is issued on login.
2
Build a developer profile
Add skills, bio, and GitHub link. Profile is visible to all users so project owners can evaluate applicants.
3
Post or browse project ideas
Any logged-in user can post a project with title, description, tech stack, and team size. All projects are searchable and filterable.
4
Request to join projects
Developers send a join request with a message. The project owner sees all requests and can accept or reject each one.
5
Collaborate with real-time chat
Accepted team members get access to the project chat room. Messages are sent instantly via Socket.io and saved to MongoDB.

Authentication system

1
Register
User submits name, email, password → bcrypt hashes password → saved to MongoDB → JWT token returned
2
Login
Email + password submitted → bcrypt compares hash → JWT token issued → stored in localStorage
3
Protected routes
Every request sends JWT in Authorization header → server verifies token → grants or denies access

Real-time chat system

1
Connect
Socket.io connects to server using JWT token for authentication when user opens chat
2
Join room
User joins a socket room named after the project ID. Only project members can access
3
Send message
Message emitted to server → saved to MongoDB → broadcast to all users in the room instantly

How a user registers and logs in

1
User fills register form on frontend
React collects name, email, password and calls POST /api/auth/register via Axios
2
Backend receives the request
Express validates fields, checks if email exists, hashes password with bcrypt cost factor 12
3
User saved to MongoDB
Mongoose creates a new User document with hashed password stored
4
JWT token issued and returned
Backend generates a 30-day JWT token and sends it back with user data
5
Frontend stores token
AuthContext saves user and token to localStorage. All future API requests attach this token automatically

How a project gets created and joined

1
Owner fills Create Project form
Title, description, tech stack, max team size submitted to POST /api/projects with JWT token
2
Project saved with owner reference
MongoDB stores project with owner field pointing to the user's ID
3
Another developer sends join request
POST /api/join-requests with projectId and message. System checks they are not already a member
4
Owner accepts or rejects
PUT /api/join-requests/:id with status accepted or rejected. On accept, user added to project collaborators
5
Team can now chat
Both owner and collaborators can access the real-time chat room for that project
